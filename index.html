Real AWS Project Behind the Game

The game explains the same skills used in a secure AWS operations project: private EC2 access, VPC networking, monitoring, alerting, and incident-response thinking.

VPC Networking

Public and private subnets separate internet-facing resources from internal workloads.

Private EC2

The internal server has no public IP and should not be directly exposed to the internet.

Session Manager

Administrators can access the private EC2 instance through IAM and AWS Systems Manager.

CloudWatch + SNS

CPU alarms detect high utilization and send email notifications through SNS.

Planned AWS workflow

  1. Create a custom VPC with public and private subnets.
  2. Attach an Internet Gateway to the public route table.
  3. Use a NAT Gateway for private subnet outbound internet access.
  4. Launch a private EC2 instance without a public IP address.
  5. Attach an IAM role with Systems Manager permissions.
  6. Connect using Session Manager instead of opening SSH to the world.
  7. Create a CloudWatch CPU alarm and connect it to an SNS email topic.
  8. Run a stress test, confirm the alarm, and document screenshots.

Cloud support mindset

Troubleshooting Notes

Private EC2 cannot reach internet

Check private route table, NAT Gateway status, public subnet route to IGW, SG outbound, and NACL rules.

Session Manager unavailable

Check IAM role, SSM Agent, instance profile, NAT Gateway, or SSM VPC endpoints.

No SNS email received

Confirm the SNS subscription, alarm action, alarm state, and spam folder.
